# Project Rules

## 우선순위

1. Product Fidelity
2. Evidence Validation
3. Realism
4. Graphic Quality

그래픽 품질은 제품 충실도와 근거 검증을 넘을 수 없다.

## DB 우선

- 제품군 DB를 최우선 기준으로 사용한다.
- 제품군 DB와 AI 일반 지식이 충돌하면 제품군 DB를 따른다.
- 제품군 DB와 입력 데이터가 충돌하면 운영 기준에 따라 판단한다.
- `compareTarget`은 제품군 판별의 주 기준이 아니다.

## 근거 우선

- AI 추론보다 입력값, 제조사 자료, 제품군 DB, 운영 기준을 우선한다.
- 근거 없는 수치 생성 금지
- 근거 없는 인증 생성 금지
- 근거 없는 등급 생성 금지
- 근거 없는 내구성, 수명, 강도 비교 금지
- 근거 없는 품질 우열 비교 금지

## HTML과 이미지 분리

- HTML 문장을 이미지에 반복하지 않는다.
- HTML 구매 체크포인트와 유사한 문구를 이미지에 억지로 넣지 않는다.
- HTML은 설명과 확인 사항을 담당한다.
- 인포그래픽은 비교, 구조, 질감처럼 시각화 가치가 있는 정보만 담당한다.

## A타입 규칙

- A타입은 비교 전용이다.
- `compareTarget`에 `VS`가 있으면 좌우 라벨을 코드에서 분리한다.
- 비교 방향은 AI가 추론하지 않는다.
- 운영 기준이 있는 비교는 운영 기준을 따른다.
- 구조 차이가 없으면 A타입 3단을 생성하지 않는다.
- 구조 차이가 있는 경우에만 3단 좌우 구조 비교를 유지한다.

## A타입 3단 생성 금지 대상

- PLYWOOD 원산지 비교
- MDF 동일 제품군 내 브랜드 또는 제조사 비교
- 같은 제품군 내 원산지, 브랜드, 표면, 선별 기준 중심 비교
- 실제 내부 구조가 동일한 비교

이 경우 최종 레이아웃은 1단과 2단으로 끝낸다.

## A타입 3단 유지 대상

- DECK_BOARD 시공 방식 비교
- PF VS XPS
- XPS VS EPS
- 서로 다른 제품군 비교
- 실제 코어, 단면, 체결 구조가 다른 비교

## B타입 규칙

- B타입은 단일 제품 구조 설명 전용이다.
- 제품군별 구조 템플릿을 따른다.
- 다른 제품군 구조를 섞지 않는다.
- A타입 비교 내용을 반복하지 않는다.

## C타입 규칙

- C타입은 표면 질감 설명 전용이다.
- 표면 상태와 질감 차이를 중심으로 구성한다.
- 구조 설명이나 비교 우열로 확장하지 않는다.

## DECK_BOARD 규칙

- Hidden Clip 시공과 상부 피스 노출 시공을 혼용하지 않는다.
- 상부 피스 노출 영역에는 Hidden Clip, 측면 홈가공, 클립 부품을 생성하지 않는다.
- Hidden Clip 영역에는 상판 피스 노출을 생성하지 않는다.
- 실제 판매 가능한 부품과 위치만 표현한다.

## PLYWOOD 원산지 비교 규칙

- 베트남산과 동남아산 비교 방향을 고정한다.
- 고정 데이터의 라벨과 bullet을 그대로 사용한다.
- AI가 문구를 재작성하거나 방향을 바꾸지 않는다.
- 입력에 없는 수종명을 생성하지 않는다.
- 라디아타파인, 프리미엄 수종, 균일 선별, 편차 최소화 같은 근거 없는 표현을 금지한다.

## 금지 표현

- 최고급
- 프리미엄
- 추천
- 가성비
- 보장
- 근거 없는 고품질
- 근거 없는 인증
- 근거 없는 수치
- 근거 없는 품질 우열

## 최소 수정 원칙

- 승인된 파일과 함수만 수정한다.
- 안정 동작을 불필요하게 바꾸지 않는다.
- 문제 원인이 있는 최소 위치만 수정한다.
- HTML/CSS/Schema 변경은 별도 승인 없이는 하지 않는다.
- Apps Script Push, Commit, Git Push는 사용자가 명시할 때만 수행한다.

## 운영 규칙

- 조사 요청에서는 코드 수정하지 않는다.
- 승인 요청 후에만 수정한다.
- context.md는 작업 이력 원본으로 유지한다.
- 운영 기준은 별도 문서로 정리한다.
- upstream에는 사용자가 명시하지 않는 한 push하지 않는다.
