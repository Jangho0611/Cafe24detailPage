## 추가 작업

### X열 HTML 생성 로직 개선
- X열 HTML 생성 로직을 구매자가 읽기 쉬운 상세페이지 구조로 보완했다.
- H2는 제품명만 출력하도록 변경했다.
- 규격, 두께, 등급, 제조사 등은 기존처럼 스펙 표에서만 표시하도록 유지했다.

### define 문체 개선
- define 문장은 제품명 반복을 피하고 제품 정의와 대표 사용처 중심으로 작성하도록 개선했다.
- 광고성 표현과 AI 문체가 들어가지 않도록 금지 표현을 강화했다.
- `적합합니다`, `프리미엄`, `뛰어난`, `고급`, `최적` 등 과장 표현을 제한했다.

### ds-reason 생성 방식 변경
- ds-reason 영역을 AI가 생성한 `reason1~3` 출력 방식에서 `buildDefaultNotes(data)` 기반 출력 방식으로 변경했다.
- `content.notes`와 `content.reason1~3`는 실제 ds-reason 출력에 사용하지 않도록 정리했다.
- ds-reason은 제품 설명이 아니라 구매자가 참고하는 추가 안내문 영역으로 조정했다.

### notes 후처리 및 문장 조립 개선
- `sanitizeNoteText()` 후처리를 추가해 광고성 원문 표현을 중립 표현으로 정제했다.
- 시트 입력값에 광고성 표현이 있어도 ds-reason에는 그대로 출력되지 않도록 했다.
- use1/use2 조합 시 조사 오류가 발생하지 않도록 `과` 대신 `및` 연결 방식으로 수정했다.
- 예: `인테리어 가구 심재 및 벽체 바탕재 작업에 사용됩니다.`

### 제품군별 안내문 분기
- 제품군별 안내문 분기를 추가했다.
- 합판, MDF, 석고보드, 각재, PF보드 계열에 따라 기본 안내문이 다르게 출력되도록 했다.
- 구조 설명은 `기준으로 확인합니다` 문체를 줄이고 제품 구조 설명형 문장으로 조정했다.

### 시트 재생성 테스트
- 일반합판, 자작합판, MDF, 소송각재, GS자이 일반석고보드, LX PF보드 행을 대상으로 X열 HTML 재생성 테스트를 진행했다.
- 일반합판 ds-reason의 use1/use2 조사 오류가 `및` 연결 방식으로 수정된 것을 확인했다.
- 최종 `clasp push` 후 시트에서 일반합판 X열 삭제 및 재생성 검수를 완료했다.

## ds-reason 읽는 흐름 개선 1차

### 제품군별 기본 안내문 조정
- MDF 기본 안내문 순서를 `노출 마감 → 도장 → 재단 → 재고` 흐름으로 조정했다.
- PF보드 기본 안내문 순서를 `두께/마감 조건 → 단열 시공 조건 → 연결 부위/마감 방법 → 재고` 흐름으로 조정했다.

### MDF/PF보드 문장 깨짐 보정
- MDF emphasis 원문이 fallback 문장으로 깨져 출력되지 않도록 도장 및 필름 래핑 관련 고정 안내문으로 치환했다.
- PF보드 structure 원문이 길게 이어질 때 고정 구조 문장으로 치환했다.
- PF보드 emphasis 원문에 성능 단정형 표현이 포함될 때 제조사 자료 기준 확인 문장으로 치환했다.

### 시트 재생성 테스트
- MDF와 LX PF보드 행의 X열을 수동 삭제 후 재생성해 치환 문장 출력과 깨진 문장 제거를 확인했다.
- MDF 도장 관련 중복은 향후 주제 중복 제거 과제로 남겼다.
