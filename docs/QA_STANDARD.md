# QA Standard

## Design Principle

인포그래픽은 상세페이지를 보완하는 B2B 정보 자료다.

- 광고 배너가 아니다.
- HTML 문장 요약본이 아니다.
- 제품 구조와 비교 정보를 빠르게 이해시키는 보조 이미지다.

## Evidence Validation

### PASS

- 입력값, 제조사 자료, 제품군 DB, 운영 기준에 있는 정보만 사용한다.
- 수치, 인증, 등급이 근거와 함께 사용된다.
- 근거가 없으면 섹션을 만들지 않는다.

### WARNING

- “확인 필요” 수준의 정보를 시각적으로 강조한다.
- 수치가 없는데 수치형 영역처럼 보이는 레이아웃을 만든다.
- 비교 문구가 입력값보다 조금 넓게 해석된다.

### FAIL

- 근거 없는 수치 생성
- 근거 없는 인증 생성
- 근거 없는 등급 생성
- 근거 없는 품질 우열 생성
- 입력에 없는 수종명 생성

## Product Fidelity

### PASS

- 제품군 DB의 구조와 일치한다.
- 비교 대상별 구조가 섞이지 않는다.
- 제품군별 금지 구조가 생성되지 않는다.

### WARNING

- 구조는 대체로 맞지만 일부 부품 위치가 애매하다.
- 제품군 구조가 너무 일반화되어 구분성이 약하다.

### FAIL

- 합판에 라멜 또는 집성 구조 생성
- MDF에 단판 적층 구조 생성
- PF와 XPS 코어 혼용
- Hidden Clip과 상부 피스 시공 혼용
- 실제 판매되지 않는 부품 생성

## Realism

### PASS

- 실제 제품 또는 실제 시공 구조처럼 보인다.
- 부품 위치와 비율이 현실적이다.
- 단면과 질감이 제품군에 맞다.

### WARNING

- 시각적으로는 자연스럽지만 부품 디테일이 약하다.
- 구조 설명은 맞지만 기술자료 느낌이 부족하다.

### FAIL

- 장난감 같은 부품
- 비현실적인 연결구
- 실제 시공이 불가능한 체결 구조
- 구조물이 서로 섞인 하이브리드 형태

## Graphic Quality

### PASS

- 정보 우선순위가 분명하다.
- 텍스트가 짧고 읽기 쉽다.
- 색상과 장식이 절제되어 있다.
- B2B 자료 스타일을 유지한다.

### WARNING

- 정보는 맞지만 카드 수가 많아 복잡하다.
- 텍스트가 조금 길다.
- 섹션 간 반복이 일부 있다.

### FAIL

- 광고 배너처럼 보인다.
- 상세설명 HTML 문장을 반복한다.
- 텍스트가 흐리거나 너무 많다.
- 빈 정보를 억지로 채운다.

## Priority

문제가 충돌하면 아래 순서로 판단한다.

1. Product Fidelity
2. Evidence Validation
3. Realism
4. Graphic Quality

예쁘지만 구조가 틀린 이미지는 FAIL이다.

## PASS / WARNING / FAIL

### PASS

- 제품군 구조가 맞다.
- 근거 없는 정보가 없다.
- HTML과 중복되지 않는다.
- 타입별 역할에 맞다.

### WARNING

- 핵심 정보는 맞지만 표현이 다소 일반적이다.
- 그래픽 품질이 낮지만 제품 정보는 맞다.
- 비교 포인트가 일부 중복된다.

### FAIL

- 제품군 구조가 틀렸다.
- 비교 방향이 뒤집혔다.
- 근거 없는 수치, 인증, 등급이 생성됐다.
- A타입에서 구조 차이가 없는 비교에 3단 구조 비교를 만들었다.
- DECK_BOARD 시공 방식이 혼용됐다.

## 제품군별 규칙

### PLYWOOD

- 단판 적층 구조만 사용한다.
- 라멜, 집성, 핑거조인트 구조 금지
- 원산지 비교는 3단 구조 비교를 생성하지 않는다.
- 베트남산과 동남아산 비교 방향을 운영 기준대로 유지한다.

### MDF

- 목재 섬유 압축 구조로 표현한다.
- 단판 적층, 라멜 집성 금지
- 동일 제품군 브랜드 비교는 구조 비교를 생성하지 않는다.

### DECK_BOARD

- 목재 결, 측면 단면, 피스 고정 위치, 데크 간격 중심
- Hidden Clip과 상부 피스 노출 시공 혼용 금지
- 구조 차이가 있는 시공 방식 비교는 3단 구조 비교 유지

### PF

- Facing / PF Foam Core / Facing 구조
- XPS, EPS, 글라스울 구조와 혼용 금지
- XPS와 비교할 경우 구조 비교 유지

### XPS

- Closed Cell, 압출 발포, XPS Core 중심
- PF Core, EPS 비드 구조, Glass Wool Fiber 금지
- PF 또는 EPS와 비교할 경우 구조 비교 유지

### EPS

- 비드 발포 입자 구조 중심
- 압출 발포 구조와 혼용 금지

### 글라스울 / 미네랄울

- 섬유 매트 구조 중심
- 발포 코어 구조 금지

## 신규 제품군 추가 규칙

신규 제품군을 추가할 때는 아래 항목을 먼저 정의한다.

- 제품군 이름
- 매칭 키워드
- keyValue 기준
- structure 기준
- emphasis 후보
- use 후보
- infographicStructure 기준
- infographicKeywords
- forbiddenKeywords
- priorityMetrics
- A/B/C 타입에서의 역할
- 구조 비교 유지 또는 생략 기준

## 운영 규칙

- 조사 단계에서는 수정하지 않는다.
- 승인된 파일과 범위만 수정한다.
- Apps Script Push는 사용자가 요청할 때만 수행한다.
- Commit과 Git Push는 사용자가 요청할 때만 수행한다.
- context.md는 작업 이력 원본으로 유지한다.
- QA 결과는 다음 Sprint의 수정 근거로 사용한다.
