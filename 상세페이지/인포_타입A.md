# 인포_타입A.md

## 0. 문서 목적

이 문서는 **카페24 상품 상세설명 영역에 삽입할 A타입 인포그래픽 이미지 생성 기준**이다.

A타입은 **수치형 인포그래픽**이다.  
수치가 구매 판단에 중요한 건축자재에 적용한다.

예시 제품:
- MDF
- 단열재
- 석고보드
- 그라스울
- PF보드
- 방화/차음 성능이 중요한 자재

---

## 1. 핵심 목표

A타입 인포그래픽의 목표는 아래 3가지만 시각화하는 것이다.

1. 비교요소
2. 핵심수치
3. 단면구조

상세페이지 HTML 텍스트와 같은 내용을 반복하지 않는다.

---

## 2. 상세페이지 HTML과 역할 분리

### 2.1 HTML 텍스트가 담당하는 정보

상세페이지 HTML에는 아래 정보가 이미 들어간다.

- 제품명
- 규격
- 두께
- 등급
- 제조사
- 즉시출고 여부
- 이 제품이 필요한 이유
- 용도
- CTA 버튼

따라서 인포그래픽에서는 위 내용을 반복하지 않는다.

### 2.2 인포그래픽이 담당하는 정보

인포그래픽은 아래 정보만 담당한다.

| 시트 열 | 역할 |
|---|---|
| G열 비교요소 | 비교 시각화 |
| H열 핵심표현 | 수치 후킹 |
| I열 출처 | 수치 신뢰 근거 |
| J열 단면구조특징 | 단면/구조 도해 |

---

## 3. 입력 데이터

A타입 인포그래픽 생성 시 아래 데이터를 사용한다.

```json
{
  "type": "A",
  "compare_target": "{G열 비교요소}",
  "product_label": "{B열 제품명}",
  "key_value": "{H열 핵심표현}",
  "source": "{I열 출처}",
  "structure_description": "{J열 단면구조특징}"
}
```

주의:
- `{B열 제품명}`은 비교 라벨에서만 1회 사용할 수 있다.
- 제품명을 제목, 설명문, 반복 라벨로 다시 사용하지 않는다.
- 규격, 두께, 등급, 제조사, 용도는 이미지에 넣지 않는다.

---

## 4. 캔버스 기준

- 크기: 800 × 900px
- 형식: PNG
- 배경: #FFFFFF
- 세로형 3단 구성
- 카페24 상세설명 이미지 삽입용

---

## 5. 색상 기준

상세_design.md 기준 색상을 사용한다.

| 용도 | 색상 |
|---|---|
| 기본 배경 | #FFFFFF |
| 섹션 배경 | #F8F8F8 |
| 메인 | #123628 |
| 포인트 | #C9A84C |
| 텍스트 | #1C1C1C |
| 서브텍스트 | #616161 |
| 보더 | #E0E0E0 |

### 색상 사용 원칙

- #123628: 제목, 선, 주요 라벨
- #C9A84C: 핵심 숫자 강조
- #1C1C1C: 본문 텍스트
- #616161: 출처, 보조 설명
- #E0E0E0: 경계선
- #F8F8F8: 섹션 배경

주의:
- 골드는 숫자 후킹에만 사용한다.
- 새로운 색상 추가 금지.
- 그라디언트 금지.
- 과한 그림자 금지.
- 복잡한 장식 패턴 금지.

## 6. 폰트 기준

- 한글 산세리프 계열
- 가능하면 Noto Sans KR 또는 Pretendard 느낌
- 제목: Bold
- 본문: Regular
- 한글 가독성 최우선

이미지 안 텍스트는 짧게 구성한다.

권장:
- 2~6자 라벨
- 최대 12자 이내 문구

금지:
- 긴 문장
- 영어 라벨
- SECTION, S1, S2, S3 표기

## 7. 레이아웃 구조

A타입은 세로 3단 구성이다.

```text
┌──────────────────────────────┐
│ 1단: 비교                     │
├──────────────────────────────┤
│ 2단: 핵심 수치                │
├──────────────────────────────┤
│ 3단: 단면 구조                │
└──────────────────────────────┘
```

단, 기계적인 박스 나열처럼 보이면 안 된다.  
첫 번째 샘플 이미지처럼 **제품 질감, 비교 구조, 깊이감**은 허용한다.

---

## 8. 1단: 비교 영역

### 목적

비교대상과 현재 제품의 차이를 한눈에 보여준다.

### 구성

- 좌우 비교 구조
- 왼쪽: `{G열 비교요소}`
- 오른쪽: `{B열 제품명}`
- 중앙 또는 하단에 차이 키워드 2~3개

### 표시 가능 정보

- 비교 대상 라벨
- 제품 라벨
- 차이 키워드

예시:
- 조직 균일성
- 표면 안정성
- 가공 편차 감소

### 금지

- 제품 설명문 반복
- 규격, 두께, 등급 표기
- 용도 표기
- 긴 문장

---

## 9. 2단: 핵심 수치 영역

### 목적

구매자가 기억해야 할 핵심 수치를 크게 보여준다.

### 구성

- 중앙 대형 숫자
- 단위
- 출처

### 표시 예시

```text
600~800 kg/㎥
출처: KS F 3200
```

### 강조 규칙

- 숫자만 #C9A84C 골드로 강조
- 단위는 텍스트색 또는 서브텍스트색 사용
- 출처는 작게 표시

### 금지

- 수치와 무관한 설명 추가
- 과도한 장식
- 임의 수치 생성

---

## 10. 3단: 단면 구조 영역

### 목적

제품의 내부 구조를 글이 아니라 도해로 이해시킨다.

### 구성

- 단면 도해 또는 레이어 구조
- 핵심 구조 키워드 3개 이하

예시:
- 균일 섬유
- 압축 성형
- 매끈한 표면

### 디자인 방향

- 단순한 단면 이미지
- 구조를 이해할 수 있는 확대 표현 허용
- 제품 질감 표현 허용
- 과도한 실사 표현보다 정보 도해 우선

### 금지

- 긴 설명문
- 용도 반복
- 규격/두께/등급 반복

---

## 11. 절대 금지 사항

다음은 이미지에 넣지 않는다.

- 규격
- 두께
- 등급
- 제조사
- 용도
- 상세설명 HTML 문장 반복
- 강조포인트 원문 그대로 사용
- SECTION, S1, S2, S3 라벨
- 영어 라벨
- 광고 배너식 과장 문구
- 과도한 색상
- 복잡한 배경 장식

제품명은 **비교 라벨에서만 1회 허용**한다.

## 12. 하이브리드 디자인 원칙

이번 테스트 결과, 너무 통제적인 프롬프트는 이미지 품질을 떨어뜨린다.  
따라서 아래 원칙을 적용한다.

### 유지할 것

- 첫 번째 샘플 이미지의 시각적 완성도
- 제품 질감 표현
- 비교 구조의 직관성
- 깊이감 있는 도해
- 정보가 한눈에 들어오는 구성

### 제한할 것

- 텍스트 중복
- 제품 스펙 반복
- 장식 과다
- 색상 과다
- 긴 문장

### 결론

이미지 생성 모델에는 **디자인 자유도**를 일부 허용하고,  
중복 금지와 핵심 정보만 강하게 통제한다.

- 등급 표시 금지
- 제조사 표시 금지

## 16. 실패 시 수정 기준

### 문제: 이미지가 너무 밋밋하다
수정:
- 제품 질감 표현 허용
- 단면 확대 표현 강화
- 비교 영역에 실제 자재 느낌 추가

### 문제: 텍스트가 너무 많다
수정:
- 문장을 키워드로 축소
- 각 영역 텍스트 3줄 이하로 제한

### 문제: 색상이 많다
수정:
- 딥그린, 골드, 회색계열만 남긴다

### 문제: 상세 HTML과 중복된다
수정:
- 규격/두께/등급/용도 제거
- 강조포인트 문장 제거
- G/H/I/J열 중심으로 재생성

### 문제: 한글이 깨진다
수정:
- 텍스트를 더 짧게 줄인다
- 이미지 API로 배경/도해만 생성하고, 한글은 Satori/HTML 후처리로 얹는다

## 17. 감독관 최종 판단 기준

A타입 인포그래픽은 아래 3가지만 성공하면 된다.
