# 인포_타입B.md

## 0. 문서 목적

이 문서는 **카페24 상품 상세설명 영역에 삽입할 B타입 인포그래픽 이미지 생성 기준**이다.

B타입은 **구조형 인포그래픽**이다.  
수치보다 **단면, 적층, 코팅층, 표면층, 코어 구조, 접착 방식**이 구매 판단에 중요한 제품에 적용한다.

예시 제품:
- 내수합판
- 태고합판
- 코아합판
- 백색 코팅 합판
- 오크 미장합판
- 집성목
- 마감용 구조재
- 한솔참마루 시리즈

---

## 1. 핵심 목표

B타입 인포그래픽의 목표는 아래 3가지를 시각화하는 것이다.

1. 단면 구조
2. 구조 상세 확대
3. 핵심 구조 키워드

A타입이 **수치 후킹**이라면, B타입은 **구조 이해**가 핵심이다.

---

## 2. 상세페이지 HTML과 역할 분리

### 2.1 HTML 텍스트가 담당하는 정보

상세페이지 HTML에는 아래 정보가 이미 들어간다.

- 제품명
- 규격
- 두께
- 등급
- 제조사
- 즉시출고 여부
- 이 제품이 필요한 이유
- 용도
- CTA 버튼

따라서 인포그래픽에서는 위 내용을 반복하지 않는다.

### 2.2 인포그래픽이 담당하는 정보

B타입 인포그래픽은 아래 정보만 담당한다.

| 시트 열 | 역할 |
|---|---|
| G열 비교요소 | 시트 호환용으로 유지, B타입에서는 완전히 무시 |
| H열 핵심표현 | 구조/질감/표면 특성 키워드 |
| J열 단면구조특징 | 단면 도해의 핵심 근거 |
| K열 강조포인트 | 원문 사용 금지, 구조 키워드 도출에만 참고 |

주의:
- K열 강조포인트 문장은 그대로 쓰지 않는다.
- L/M열 용도는 사용하지 않는다.
- 규격/두께/등급/제조사는 사용하지 않는다.

---

## 3. 입력 데이터

B타입 인포그래픽 생성 시 아래 데이터를 사용한다.

```json
{
  "type": "B",
  "product_label": "{B열 제품명}",
  "structure_keyword": "{H열 핵심표현}",
  "structure_description": "{J열 단면구조특징}",
  "reference_note": "{K열 강조포인트에서 구조 관련 핵심만 추출}"
}
```

주의:
- G열 비교요소와 `compare_target`은 B타입 프롬프트에 전달하지 않는다.
- B타입은 반드시 단일 제품의 구조만 표현한다.
- 다른 제품명, 경쟁 제품명, 대체 소재명은 표시하지 않는다.
- 제품명을 제목, 설명문, 반복 라벨로 사용하지 않는다.
- 규격, 두께, 등급, 제조사, 용도는 이미지에 넣지 않는다.

- 크기: 800 × 900px
- 형식: PNG
- 배경: #FFFFFF
- 세로형 3단 구성
- 카페24 상세설명 이미지 삽입용

## 5. 색상 기준

상세_design.md 기준 색상을 사용한다.

| 용도 | 색상 |
|---|---|
| 기본 배경 | #FFFFFF |
| 섹션 배경 | #F8F8F8 |
| 메인 | #123628 |
| 포인트 | #C9A84C |
| 텍스트 | #1C1C1C |
| 서브텍스트 | #616161 |
| 보더 | #E0E0E0 |

### 색상 사용 원칙

- #123628: 제목, 선, 주요 라벨, 구조 포인트
- #C9A84C: 핵심 구조 포인트, 번호, 강조 아이콘
- #1C1C1C: 본문 텍스트
- #616161: 보조 설명
- #E0E0E0: 경계선
- #F8F8F8: 섹션 배경

주의:
- 골드는 구조의 핵심 포인트에만 사용한다.
- 수치가 없으면 숫자 후킹을 만들지 않는다.
- 새로운 색상 추가 금지.
- 그라디언트 금지.
- 과한 그림자 금지.
- 복잡한 장식 패턴 금지.

## 6. 폰트 기준

- 한글 산세리프 계열
- 가능하면 Noto Sans KR 또는 Pretendard 느낌
- 제목: Bold
- 본문: Regular
- 한글 가독성 최우선

이미지 안 텍스트는 짧게 구성한다.

권장:
- 2~8자 라벨
- 최대 12자 이내 문구
- 구조 키워드 중심

금지:
- 긴 설명문
- 영어 라벨
- SECTION, S1, S2, S3 표기
- Application, Feature, Layer 같은 영어 표기

## 7. 레이아웃 구조

B타입은 세로 3단 구성이다.

```text
┌──────────────────────────────┐
│ 1단: 단면 구조 메인           │
├──────────────────────────────┤
│ 2단: 구조 상세 확대 영역       │
├──────────────────────────────┤
│ 3단: 핵심 구조 키워드          │
└──────────────────────────────┘
```

A타입과 달리, B타입은 **단면 구조가 가장 중요하므로 1단에 크게 배치**한다.

---

## 8. 1단: 단면 구조 메인

### 목적

제품의 내부 구조를 가장 먼저 이해시킨다.

### 구성

- 큰 단면 도해 또는 분해도
- 레이어, 코어, 표면층, 접착층 등을 단순하게 표현
- 오른쪽 또는 주변에 짧은 라벨 3개 이하

### 표시 가능 정보

- 단면 구조 키워드
- 레이어 이름
- 구조 특성

예시:
- 교차 적층
- 균일 단판
- 표면층
- 코어층
- 코팅층
- 접착층
- 균일 코어
- 안정 구조

### 금지

- 제품 설명문 반복
- 규격, 두께, 등급 표기
- 용도 표기
- 긴 문장

---

## 9. 2단: 구조 상세 확대 영역

### 목적

상단 메인 단면 구조를 확대/분해하여 동일 제품 내부 구조를 상세하게 보여준다.

### 구성

- 코어 확대뷰
- 표면층 확대뷰
- 접착층 확대뷰
- 동일 제품 구조 확대뷰 3개
- 확대 이미지 중심, 제품명 반복 없이

### 중요

- 반드시 동일 제품만 사용
- 확대창은 상단 구조에서 가져온 세부만 보여준다

---

## 10. 3단: 핵심 구조 키워드

### 목적

1단과 2단에서 보여준 단일 제품 구조의 핵심 장점을 정리한다.

### 구성

- 3개 또는 4개 아이콘/키워드 카드
- 각 카드에는 짧은 구조 키워드만 사용
- 단일 제품의 구조 키워드 카드만 사용
- 긴 설명은 넣지 않는다

### 표시 예시

구조형 합판:
- 균일 단판
- 교차 적층
- 깨끗한 단면
- 정밀 가공

표면 코팅 합판:
- 투명 코팅
- 표면 보호
- 도장 공정 절감
- 균일 광택

백색 코팅 합판:
- 백색 표면
- 필름 라미네이팅
- 즉시 마감
- 매끄러운 면

집성목:
- 핑거 접합
- 원목 결감
- 안정 집성
- 균일 판면

### 금지

- 상세설명 문장 반복
- 용도 문장 반복
- 과도한 아이콘 사용
- 영어 라벨

---

## 11. 절대 금지 사항

다음은 이미지에 넣지 않는다.

- 두 제품을 나란히 배치하는 구성
- 화면을 좌우로 분할하는 레이아웃
- 다른 제품 표시
- 경쟁 제품 표시
- 대체 소재명 표시
- 규격
- 두께
- 등급
- 제조사
- 용도
- 상세설명 HTML 문장 반복
- 강조포인트 원문 그대로 사용
- SECTION, S1, S2, S3 라벨
- 영어 라벨
- 광고 배너식 과장 문구
- 과도한 색상
- 복잡한 배경 장식

제품명은 이미지 내부 제목이나 설명문으로 반복하지 않는다.

---

## 12. B타입 하이브리드 디자인 원칙

B타입은 구조형이므로 이미지 생성 모델의 시각 표현 능력을 일부 활용한다.

### 유지할 것

- 제품 질감 표현
- 단면 구조의 깊이감
- 레이어 구조의 입체감
- 자재별 표면 질감
- 첫 번째 샘플 이미지 수준의 완성도

### 제한할 것

- 텍스트 중복
- 제품 스펙 반복
- 색상 과다
- 설명문 과다
- 과장된 광고 포스터 느낌

### 결론

B타입은 **텍스트로 설명하지 말고 구조를 보여주는 이미지**여야 한다.  
단면과 레이어가 잘 보이면 성공이다.

## 13. ChatGPT API 이미지 생성용 프롬프트

아래 프롬프트를 기본 템플릿으로 사용한다.

```text
건축자재 B2B 쇼핑몰 상세페이지용 한국어 인포그래픽 이미지를 생성하라.

이 이미지는 카페24 상품 상세설명 HTML 안에 삽입되는 보조 이미지다.
광고 포스터가 아니라, 제품의 구조를 이해시키는 실무형 인포그래픽으로 만든다.

목표:
- 상세설명 HTML 텍스트와 중복 금지
- 단면 구조, 구조 상세 확대, 핵심 구조 키워드만 시각화
- 한눈에 단면 구조가 이해되는 B2B 자료 스타일
- 한글 텍스트는 선명하고 정확하게 표시

캔버스:
- 800 x 900px
- 배경 #FFFFFF
- 세로형 3단 구성

색상:

- 섹션 배경 #F8F8F8

- 보더 #E0E0E0

디자인 방향:
- 고급 B2B 인포그래픽 스타일
- 제품 질감과 단면 구조를 자연스럽게 표현
- 단면 구조가 한눈에 보이도록 구성
- 색상은 절제
- 골드는 핵심 구조 포인트에만 사용
- 그라디언트와 과한 그림자는 사용하지 않음
- 장식보다 정보 전달 우선

1단: 단면 구조 메인

- 레이어, 코어, 표면층, 접착층 등 구조 포인트를 시각화

- 긴 설명문 금지

- surface layer zoom
- core layer zoom
- adhesive layer zoom
- no product names in this section
- no title bars in this section
- each zoom window must show magnified details from the top structure only
- 동일 제품 구조 확대뷰 3개
- 표면층 확대

- 3개 또는 4개 카드형 키워드 구성
- "{H열 핵심표현}"과 "{J열 단면구조특징}"에서 구조 키워드만 추출
- 단일 제품의 구조 키워드 카드만 사용
- 아이콘은 단순하게
- 긴 설명문 금지

절대 금지:
- 두 제품을 나란히 배치 금지
- 화면을 좌우로 분할하는 레이아웃 금지
- 단일 제품의 단면 구조만 표현
- 하단은 단일 제품의 구조 키워드 카드만 사용
- Create infographic for ONE product only.
- Do not display any alternative material.
- The infographic must describe only one product.
- Middle section must contain enlarged structure detail views of the same product.
- Do not split the layout into left and right product sections.
- 규격 표시 금지
- 두께 표시 금지
- 등급 표시 금지
- 제조사 표시 금지
- 용도 표시 금지
- 제품명을 제목이나 설명문으로 반복 금지
- 상세설명 HTML 문장 반복 금지
- SECTION, S1, S2, S3 같은 라벨 금지
```

## 15. 실패 시 수정 기준

### 문제: 텍스트가 너무 많다
수정:
- 문장을 키워드로 축소
- 각 영역 텍스트 3줄 이하로 제한

### 문제: 구조가 잘 안 보인다
수정:
- 1단 단면 구조를 더 크게 배치
- 라벨을 줄이고 도해 비중을 늘린다

### 문제: 수치형처럼 보인다
수정:
- 핵심 수치 영역을 제거
- 단면/레이어 중심으로 재생성

### 문제: 색상이 많다
수정:
- 딥그린, 골드, 회색계열만 남긴다

### 문제: 상세 HTML과 중복된다
수정:
- 규격/두께/등급/용도 제거
- 강조포인트 문장 제거
- H/J/K열 중심으로 재생성

### 문제: 한글이 깨진다
수정:
- 텍스트를 더 짧게 줄인다
- 이미지 API로 배경/도해만 생성하고, 한글은 Satori/HTML 후처리로 얹는다

## 16. 감독관 최종 판단 기준

B타입 인포그래픽은 아래 2가지만 성공하면 된다.

1. 단면 구조가 가장 먼저 보이는가
2. 핵심 구조 키워드가 짧고 명확한가

이 2가지가 되면 성공이다.
